# DomaDrip

Passive XP farming for Doma tokenized domain holders.

## Features

- **Connect Wallet**: Supports Wagmi + Viem on Doma Mainnet.
- **Fetch Tokens**: Automatically fetches your .com, .ai, .xyz tokens from Doma Explorer.
- **Start XP**: Activate farming for your tokens.
- **XP Drip**: 1$ Value = 10 XP/day. +20% Bonus for >1000 tokens.
- **Leaderboard**: Track top farmers with real-time stats.

## Setup

1. **Clone & Install**
   ```bash
   git clone <repo>
   cd doma-drip
   npm install
   ```

2. **Environment Variables**
   Copy `.env.example` to `.env.local` and fill in your Supabase credentials.
   ```bash
   cp .env.example .env.local
   ```

3. **Supabase Database Setup**
   Run the following SQL in your Supabase SQL Editor:

   ```sql
   -- Create users table
   create table users (
     wallet text primary key,
     nickname text,
     created_at timestamp with time zone default timezone('utc'::text, now()) not null
   );

   -- Create activations table
   create table activations (
     id bigint generated by default as identity primary key,
     wallet text references users(wallet),
     token_symbol text not null,
     activated_at timestamp with time zone default timezone('utc'::text, now()) not null,
     balance_at_activation numeric not null,
     price_at_activation numeric not null,
     last_claimed_at timestamp with time zone,
     current_xp numeric default 0,
     unique(wallet, token_symbol)
   );

   -- Enable Row Level Security (RLS) if needed, or leave public for MVP
   alter table users enable row level security;
   alter table activations enable row level security;

   -- Policy for public read (MVP)
   create policy "Public users are viewable by everyone." on users for select using (true);
   create policy "Public activations are viewable by everyone." on activations for select using (true);

   -- Policy for insert/update (MVP - allow anon for demo, in prod use auth)
   create policy "Users can insert their own profile." on users for insert with check (true);
   create policy "Users can update their own profile." on users for update using (true);
   create policy "Users can insert activations." on activations for insert with check (true);
   ```

4. **Run Development Server**
   ```bash
   npm run dev
   ```

## Tech Stack

- **Framework**: Next.js 15 (App Router)
- **Styling**: Tailwind CSS v4 + Framer Motion
- **Web3**: Wagmi + Viem + TanStack Query
- **Database**: Supabase
- **Chain**: Doma Mainnet (ID: 97477)
